Database Design Document
Project: Flood Prediction and Early Warning System
Version: 1.0
Date: June 2026

1. Database Schema Overview
This database design is tailored for the Flood Prediction and Early Warning System. It supports user administration, telemetry storage from weather stations (historical and real-time), machine learning model metadata tracking, and prediction logs with automated alerts.

2. Data Dictionary
Users Table
Stores information about registered users and administrators of the system.

Field Name	Data Type	Constraints	Description
user_id	INT	PK, Auto-Increment	Unique identifier for each user account.
username	VARCHAR(50)	Unique, Not Null	Unique display username chosen by the user.
email	VARCHAR(100)	Unique, Not Null	Email address used for communications and alerts.
password_hash	VARCHAR(255)	Not Null	Securely hashed user password.
role	VARCHAR(20)	Default 'User'	User role defining permissions ('Admin', 'User').
created_at	TIMESTAMP	Default CURRENT_TIMESTAMP	Timestamp indicating when the user registered.
Weather_Data Table
Stores meteorological and environmental telemetry readings collected from various location sensors.

Field Name	Data Type	Constraints	Description
data_id	INT	PK, Auto-Increment	Unique identifier for the telemetry reading.
timestamp	TIMESTAMP	Not Null	The date and time when the observation was recorded.
location	VARCHAR(100)	Not Null	The geographic name or station ID.
temperature	FLOAT	Optional	Temperature in degrees Celsius.
humidity	FLOAT	Optional	Relative humidity percentage.
precipitation	FLOAT	Not Null	Amount of rainfall recorded in millimeters (mm).
river_level	FLOAT	Not Null	Water level of the closest river in meters (m).
wind_speed	FLOAT	Optional	Wind speed in km/h.
ML_Model Table
Stores metadata, parameters, and historical accuracy scores of trained machine learning models.

Field Name	Data Type	Constraints	Description
model_id	INT	PK, Auto-Increment	Unique identifier for the trained model.
model_name	VARCHAR(50)	Not Null	Name of the algorithm (e.g. 'KNN', 'PyTorch MLP').
version	VARCHAR(10)	Not Null	Model version (e.g. 'v1.0.0').
parameters	TEXT	Optional	JSON string representing model training hyperparameters.
accuracy	FLOAT	Not Null	Validation accuracy or F1-score metric.
trained_at	TIMESTAMP	Default CURRENT_TIMESTAMP	Timestamp of model training completion.
Prediction_Result Table
Logs the outputs generated by running the ML models on the weather datasets.

Field Name	Data Type	Constraints	Description
prediction_id	INT	PK, Auto-Increment	Unique identifier for this prediction run.
data_id	INT	FK references Weather_Data	Links to the input weather telemetry data.
model_id	INT	FK references ML_Model	Links to the model that generated this prediction.
prediction_timestamp	TIMESTAMP	Default CURRENT_TIMESTAMP	Date and time the prediction was run.
flood_probability	FLOAT	Not Null	Calculated probability of flooding (0.0 to 1.0).
alert_level	VARCHAR(10)	Not Null	Resulting alert level ('Low', 'Medium', 'High').
